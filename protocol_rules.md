# FORGE Protocol Continuity Rules

This file is a lightweight continuity/support layer under [AGENTS.md](C:/FORGE/Internal%20Platform/FORGE%20CSS/AGENTS.md).

It does not define the FORGE system.
It exists to keep protocol generation continuous, summary-first, and low-friction.

## Core Standard

- Continuity over reinvention
- Prefer summaries and extracted key data before raw documents
- Read full uploaded documents only when summaries are insufficient or explicit review is required
- Use prior protocol structure and progression intent as the default starting point

## Review Order

Review source context in this order:

1. most recent protocol summary
2. original or baseline protocol summary
3. journal/check-in/adherence summary
4. coach notes summary

If summaries are incomplete, inspect only the smallest underlying source needed to resolve the gap.

## Extract Before Writing

Extract:

- prior phase
- prior split
- prior movement priorities
- prior cardio
- prior abs/core
- progression intent
- current recovery constraints
- current adherence constraints

## Document Continuity Check

Every generated protocol should begin with a **Document Continuity Check** that states:

- what summaries or sources were reviewed
- what the prior protocol was trying to accomplish
- what continuity elements must be preserved
- what current recovery or adherence constraints are active

If progression data are incomplete, state exactly:

`Insufficient progression data in uploaded documents - using conservative continuation logic.`

## Continuity Decisions

Before finalizing a protocol, explicitly determine:

- what to continue
- what to progress
- what to reduce
- what to replace
- what to hold constant

## Continuity Guardrails

- Do not reset to a generic template if a prior structure exists
- Do not remove sport specificity unless the source context justifies it
- Do not change split, cardio, or core structure without a documented reason
- When adherence or recovery drops, reduce chaos, not specificity
- When data are incomplete, prefer conservative continuation over aggressive change
